## Status: IN PROGRESS

## Codebase Patterns

- Contract schema tests use `@effect/vitest` with `it.effect(...)`, `Effect.gen(...)`, and `Schema.decodeUnknownEffect(...)`.
- Branded entity identifiers live in `packages/contracts/src/baseSchemas.ts` and are created with the local `makeEntityId` helper over `TrimmedNonEmptyString`.
- The contracts package exports schema modules through `packages/contracts/src/index.ts`; `baseSchemas.ts` changes do not require additional index wiring.
- New contract domains should live in dedicated files and be exported from `packages/contracts/src/index.ts` rather than extending unrelated schema modules.
- When additive command schemas outpace runtime support, keep the existing `OrchestrationCommand` and client RPC command unions stable and stage the expanded surface behind `ForgeCommand` until decider/engine handling is added.
- Persistence repository tests that share an `it.layer(...)` suite can observe rows created by earlier tests; scope assertions to the rows under test or use a fresh layer when isolation matters.
- Multi-repository persistence tests that must share one SQLite database should merge the repository live layers first and provide `SqlitePersistenceMemory` once to the merged layer; independently pre-providing the in-memory layer risks separate databases.
- Channel read cursors must be monotonic in persistence: stale `channel.messages-read` events may arrive later than newer reads, so `channel_reads.last_read_sequence` should only advance and must not regress on upsert.
- Snapshot freshness must include every projector whose tables are read during snapshot hydration; in particular, `ProjectionSnapshotQuery` derives `latestTurn` from `projection_turns`, so `projection.thread-turns` belongs in the required projector set alongside `projection.checkpoints`.

- 2026-04-05: Spec compliance gap in the staged Forge turn-event surface: `design/15-contracts.md` defines the provider turn events as `thread.turn-requested`, `thread.turn-started`, `thread.turn-completed`, `thread.turn-restarted`, plus a Forge-level `thread.message-sent` payload with `content`, but the contracts package still only exposed the legacy `thread.turn-start-requested` variant and legacy `thread.message-sent` payload shape. The fix added the missing Session turn/message payload schemas and Forge event variants, extended contract round-trip coverage for the spec-defined turn/message events, and hardened `ProjectionPipeline` to type-guard both the staged Forge and legacy `thread.message-sent` payload shapes without changing runtime behavior.
- 2026-04-05: Spec compliance gap in the staged Forge command surface: `design/15-contracts.md` still defines the leaf-session turn commands `thread.send-turn`, `thread.restart-turn`, and `thread.send-message`, but `packages/contracts/src/orchestration.ts` had never added those schemas to `ForgeCommand`, leaving the target contracts incomplete even though the migration plan allows the Forge surface to stage ahead of runtime support. The fix added the three command schemas, wired `thread.send-turn`/`thread.restart-turn` into the staged client command union and `thread.send-message` into the staged internal command union, and added focused round-trip coverage proving `ForgeCommand` accepts the documented variants.
- 2026-04-05: Spec compliance gap in the staged Forge lifecycle contract surface: `design/15-contracts.md` defines Forge-level session lifecycle commands/events (`thread.create` with parent/workflow/session fields, `thread.pause`, `thread.resume`, `thread.recover`, `thread.cancel`, `thread.restart`, `thread.meta-update`, plus `thread.status-changed`, `thread.completed`, `thread.failed`, `thread.cancelled`, and `thread.restarted`), but `packages/contracts/src/orchestration.ts` still staged only the additive workflow/channel/request surface on top of the legacy thread lifecycle schemas. The fix added the missing Forge lifecycle command schemas, widened the staged Forge event surface with the missing lifecycle payloads and `thread.meta-updated` description support, added focused contract round-trip coverage for the new command/event variants, and hardened `ProjectionPipeline`’s `thread.created`/`thread.archived` cases so the legacy SQL projector keeps typechecking cleanly against the wider Forge event union.
- 2026-04-05: Code consistency gap in repeated correction projection: once `thread.correct` started reusing a workflow guidance channel, the SQL projection path still upserted the channel row on every queued correction, which could rewrite `created_at` and drift from the in-memory projector. The fix taught `ProjectionPipeline` to detect an existing guidance channel row, preserve its original metadata, update only `updated_at`, and append the queued correction message at the next sequence.
- 2026-04-05: Spec compliance and integration gap in correction projection: `thread.correction-queued` was only touching thread timestamps, so queued workflow corrections were not materialized into the guidance channel projection and later retries/restarts had no persisted correction messages to read back. The fix taught `thread.correct` to reuse an existing guidance channel when present, taught the in-memory projector to materialize that guidance channel on first correction, and taught the SQL projection pipeline to persist the guidance channel plus ordered human correction messages.
- 2026-04-05: The previously reported root `bun run test` instability was not reproducible on rerun. Package-isolated `apps/server`, `packages/contracts`, and `packages/shared` test runs all passed, followed by a clean root `bun run test` (`turbo run test`) plus `bun fmt`, `bun lint`, and `bun typecheck`, so the verification blocker was cleared without code changes.
- 2026-04-05: Spec compliance gap in the read-model contract surface: `design/15-contracts.md` section 8 defines `SessionStatus`, `ForgeReadModel`, and `ForgeClientSnapshot`, but the contracts package still only exposed the widened legacy `OrchestrationReadModel`, leaving the target server/client snapshot schemas unavailable to later loops.
- 2026-04-05: Integration wiring gap in projection snapshot rehydration: `ProjectionSnapshotQuery` only hydrated the legacy project/thread surfaces and computed snapshot freshness from the old projector subset, so restart snapshots silently dropped persisted phase runs, channels, pending requests, workflow definitions, and additive thread fields even though the projection pipeline had already materialized them to SQL.
- 2026-04-05: Test coverage gap in the additive projection pipeline lifecycle paths: `ProjectionPipeline.test.ts` only exercised initial inserts for the new workflow/channel/request tables, leaving DB-backed update flows like `thread.phase-output-edited`, `channel.concluded`/`channel.closed`, and `request.stale` unverified end-to-end.
- 2026-04-05: Spec compliance gap in bootstrap lifecycle coverage: `design/15-contracts.md` defines the additive `thread.bootstrap-queued` event and queued bootstrap state, but the Forge event union and projector only handled started/completed/failed/skipped, so queued bootstrap sessions could not be represented in the read model.
- 2026-04-05: Spec compliance gap in the additive channel surface: `channel.read-messages` was defined in contracts, but the decider did not emit the matching `channel.messages-read` event and the projection pipeline never updated `channel_reads`, leaving the read-cursor path unreachable despite the schema and table existing.
- 2026-04-05: Spec compliance gap in push-event contracts: section 7 of `design/15-contracts.md` defines workflow/channel push event payload schemas, but the contracts package did not expose `WorkflowPushEvent`/`ChannelPushEvent` or their typed variants, leaving that documented schema surface unavailable to later loops.
- 2026-04-05: Error handling gap in the new JSON-backed foundation projection repositories: malformed stored JSON in workflow, phase-run, phase-output, channel-message, and interactive-request projections was being surfaced as `PersistenceSqlError` instead of the existing `PersistenceDecodeError`, which would have obscured corruption/shape regressions during projection reads.
- 2026-04-05: Reliability gap in channel read projections: `ProjectionChannelReadRepository.updateCursor` replaced `channel_reads.last_read_sequence` unconditionally, so a later stale `channel.messages-read` event could move the cursor backward and inflate unread counts after reconnects or retries.
